# THEORY.md — 数式・理論

投影幾何、台形補正、座標変換、色補正、キャリブレーションなど、数式や理論前提を扱うときの記録場所。現時点では実装対象の数式はない。

## Ownership Rule

数式は人間が所有する。AI は整合性レビュー、別案提示、実装補助、検証補助を行うが、式の正しさを独断で確定しない。

| 工程 | 担当 |
|------|------|
| 定式化（式・記号・前提を立てる） | 人間 |
| 整合性レビュー（次元・記号衝突・境界条件） | AI（提案のみ） |
| 正しさの最終判断 | 人間 |
| 実装（式↔コード対応） | AI（人間確認後） |
| 検証（既知ケース、極限、単位、実機確認） | AI+人間 |

- AI が生成・変形した式は `（未確認）` を付ける。
- 人間レビュー前の式をコードへ実装しない。
- 実装した式は [Eq ↔ Code Map](#eq--code-map) に対応を書く。

## Numerical Considerations

将来の数式実装では、次を確認する。

- 座標系の原点、軸方向、単位。
- ピクセル中心を整数座標で扱うか、半整数座標で扱うか。
- 画像変換時の補間方法。
- 浮動小数点誤差と丸め。
- 実機投影での見え方と数値上の結果の差。

## Validation

数式を実装したら、次の検証を追加する。

- [ ] 既知の単純ケースで期待座標と一致する。
- [ ] 単位行列やゼロ変換で入力が保たれる。
- [ ] 画像サイズや座標範囲の境界値で破綻しない。
- [ ] 実機投影テストを `docs/EXPERIMENTS.md` に記録する。
