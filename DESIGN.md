# DESIGN.md — 表示設計

投影ウィンドウと、将来追加する可能性がある操作 UI の表示設計メモ。現時点では実装前なので、確定仕様ではなく初期方針として扱う。

## 1. Visual Theme & Atmosphere

投影面は内容を邪魔しないことを最優先にする。背景は黒、余計な枠・影・装飾・カーソル表示を避ける。操作 UI を作る場合は、現場で素早く状態を読める静かなツール UI にする。

## 2. Color Palette & Roles

### Projection Window

| 役割 | 値 | 用途 |
|------|------|------|
| Background | `#000000` | メディア未表示時、余白、フェード時 |
| Foreground | `#ffffff` | デバッグ表示や短いステータス |
| Safe Warning | `#ffcc00` | 投影面上の警告表示が必要な場合 |
| Error | `#ff4d4f` | 表示不能などの一時エラー |

### Operator UI（将来）

| 役割 | 値 | 用途 |
|------|------|------|
| Background | `#f7f7f4` | 操作用ウィンドウ背景 |
| Surface | `#ffffff` | 入力欄、パネル |
| Text | `#1f2328` | 本文 |
| Muted Text | `#667085` | 補助情報 |
| Border | `#d0d5dd` | 境界線 |
| Accent | `#2563eb` | 主要操作 |

## 3. Typography Rules

### 3.1 和文フォント

操作 UI やデバッグ表示では、Windows 標準環境で読みやすい `Yu Gothic UI` を優先し、必要に応じて `Noto Sans JP` を使う。

### 3.2 欧文フォント

数字、座標、解像度などの表示は `Segoe UI` または `Inter` を想定する。

### 3.3 font-family 指定

```css
font-family: "Segoe UI", "Yu Gothic UI", "Noto Sans JP", sans-serif;
```

### 3.4 文字サイズ・ウェイト階層

| 用途 | size | weight | line-height |
|------|------|------|------|
| Operator heading | `18px` | `600` | `1.4` |
| Operator body | `14px` | `400` | `1.6` |
| Debug overlay | `14px` | `500` | `1.4` |
| Caption | `12px` | `400` | `1.5` |

### 3.5 行間・字間

- 操作 UI の本文は `line-height: 1.6` を基準にする。
- 投影面のデバッグ表示は短い値だけにし、長文を出さない。
- letter spacing は基本 `0`。読みづらい大文字ラベルでだけ調整する。

## 4. Projection Window

- 既定背景は黒。
- フルスクリーン時はウィンドウ枠、タイトルバー、OS カーソルを投影面に出さない。
- メディア未表示や読み込み中に白いフラッシュを出さない。
- エラー表示は投影現場で必要な場合だけ短く出し、通常は操作側ログに寄せる。
- 投影ウィンドウ内の座標は左上を原点とする。

## 5. Media Fit Modes

| モード | 意味 | 用途 |
|------|------|------|
| `contain` | 縦横比を保ち、全体が収まるように表示 | 既定候補。画像全体を見せたい場合 |
| `cover` | 縦横比を保ち、画面を覆うように表示 | 余白なしで投影したい場合 |
| `stretch` | 縦横比を無視して画面に合わせる | 幾何補正済み素材など |
| `native` | 元解像度で中央または指定座標に表示 | テストパターン、座標確認 |

既定値は未確定。最初の API 設計時に決める。

## 6. Operator UI Principles（将来）

- 現場でよく使う操作を一画面に集める。
- 投影開始、停止、表示素材選択、対象ディスプレイ、フルスクリーン切替を明確に分ける。
- 危険な操作は確認を挟む。
- 実行中の投影設定を常に読めるようにする。
- 投影ウィンドウと操作 UI のウィンドウを混同しない表示名にする。

## 7. Do's and Don'ts

- ✅ 投影ウィンドウは黒背景を基本にする。
- ✅ フルスクリーンとウィンドウ表示の差を明確に扱う。
- ✅ 表示先ディスプレイ、座標、解像度をログに残せるようにする。
- ✅ テストパターン表示を最初期から用意できる設計にする。
- ❌ 投影面に不要な説明文や操作 UI を常時出さない。
- ❌ GUI バックエンド固有の見た目を公開 API に漏らさない。
- ❌ 読み込み時に白背景やちらつきが出る実装を既定にしない。

## 8. Responsive Behavior

投影ウィンドウはブラウザ的なレスポンシブ UI ではなく、指定されたディスプレイ解像度とウィンドウ geometry に厳密に従う。素材の収め方は `FitMode` で制御する。

## 9. Agent Prompt Guide

UI や表示処理を作るときは、このファイルの投影ウィンドウ方針を優先する。投影面には余計な装飾を出さず、フルスクリーン、ディスプレイ選択、ウィンドウ座標を検証しやすい構造にする。
